# РобоСклад — інтелектуальна система управління складом робототехніки

Вебзастосунок для обліку складських залишків, семантичного пошуку компонентів, управління каталогом і ролями користувачів, аналізу ризику дефіциту та підтримки рішень щодо поповнення запасів за допомогою AI-асистента **Скаут**.

Система поєднує CRUD-операції зі складом, журнал рухів запасів, графічну аналітику трендів, кошик із рекомендаціями дозамовлення, журнал запитів на поповнення та інтелектуальний чат, який працює на основі даних із PostgreSQL і Google Gemini.

## Основні можливості

- **Каталог компонентів** — пошук, фільтрація за категорією та виробником, сортування, картки товарів із фото.
- **Картка товару** — опис, технічні характеристики (`specs`), історія операцій, графік динаміки залишків за місяць, прогноз **Скаут**.
- **AI-асистент (Скаут)** — плаваючий чат на всіх сторінках: відповіді українською про склад, порівняння товарів, рекомендації щодо замовлення; fallback між моделями Gemini при перевантаженні API.
- **Кошик і запити на поповнення** — формування списку дозамовлення з урахуванням `minStockLevel`, відправка запиту в БД.
- **Адмін-панель**:
  - керування ролями (`ADMIN` / `WORKER`);
  - оновлення залишків із фіксацією причини (`REPLENISHMENT`, `ORDER`, `WRITE_OFF`);
  - масове додавання товарів;
  - журнал запитів на поповнення;
  - редагування назви, опису, ціни та характеристик товару.
- **Авторизація** — реєстрація, вхід, зашифрована cookie-сесія (`iron-session`), розмежування прав на рівні middleware і API.

## Технологічний стек

| Шар | Технології |
|-----|------------|
| Frontend | Nuxt 3, Vue 3, TypeScript, Tailwind CSS, Lucide Icons |
| Backend | Nitro (Nuxt server routes), Prisma ORM |
| База даних | PostgreSQL (Neon) |
| AI | Google Gemini API (`gemini-2.5-flash`, fallback `gemini-2.5-flash-lite`) |
| Графіки | ECharts, vue-echarts |
| Авторизація | bcrypt, iron-session |

**Потрібна версія Node.js:** `22` (див. `.nvmrc`).

## Структура проєкту

```
├── app/                    # глобальні стилі
├── components/             # UI-компоненти (кнопки, інпути, перемикач теми)
├── composables/            # useAppTheme та ін.
├── entities/               # ProductCard, StockTrendChart, useUserSession
├── features/               # форми входу/реєстрації, фільтри, AI, редагування товару
├── layouts/                # default (чат Скаут)
├── middleware/             # admin-only
├── pages/                  # маршрути застосунку
├── prisma/                 # схема, міграції, seed
├── public/catalog/         # SVG/WebP зображення каталогу
├── server/api/             # REST API (Nitro)
├── server/utils/           # prisma, session, gemini, apiError
├── shared/                 # типи API, утиліти відображення
└── widgets/                # ProductGrid, ChatSidebar, InventoryTable тощо
```

## Сторінки застосунку

| Маршрут | Опис | Доступ |
|---------|------|--------|
| `/` | Головна, огляд показників складу | Публічний |
| `/catalog` | Каталог із фільтрами | Публічний |
| `/catalog/[id]` | Деталі товару, графік, Скаут | Публічний |
| `/cart` | Кошик і запит на поповнення | Публічний |
| `/login`, `/register` | Вхід і реєстрація | Гості |
| `/admin/users` | Користувачі та ролі | Лише `ADMIN` |
| `/catalog/manage` | Запаси, інвентар, журнал запитів | Лише `ADMIN` |

## API (короткий огляд)

| Метод | Шлях | Призначення |
|-------|------|-------------|
| `POST` | `/api/auth/login`, `/register`, `/logout` | Авторизація |
| `GET` | `/api/auth/me` | Поточний користувач |
| `GET` | `/api/products` | Список товарів (пагінація, фільтри) |
| `GET` | `/api/products/:id` | Деталі товару |
| `PATCH` | `/api/products/:id` | Редагування товару (admin) |
| `GET` | `/api/products/:id/scout` | AI-прогноз по SKU |
| `GET` | `/api/stock/summary` | Зведені метрики складу |
| `POST` | `/api/ai/chat` | AI-чат Скаут |
| `POST` | `/api/replenishment-requests` | Створити запит на поповнення |
| `GET` | `/api/replenishment-requests` | Журнал запитів (admin) |
| `GET` | `/api/admin/inventory` | Інвентар для адміна |
| `GET` | `/api/admin/users` | Список користувачів |
| `PATCH` | `/api/admin/users/:id` | Зміна ролі |
| `PATCH` | `/api/components/:id/stock` | Оновлення залишку |
| `POST` | `/api/components/bulk` | Масове додавання |
| `DELETE` | `/api/components/:id` | Видалення товару |

## Налаштування середовища

1. Скопіюйте шаблон змінних:

```bash
cp .env.example .env
```

2. Заповніть `.env`:

| Змінна | Опис |
|--------|------|
| `DATABASE_URL` | Рядок підключення PostgreSQL (наприклад, Neon) |
| `NUXT_GEMINI_API_KEY` | API-ключ Google Gemini |
| `NUXT_SESSION_SECRET` | Секрет сесії, мінімум 32 символи (`openssl rand -base64 32`) |
| `NUXT_GEMINI_MODEL` | *(необовʼязково)* основна модель; за замовчуванням `gemini-2.5-flash` |

Файл `.env` **не комітиться** в репозиторій. У репозиторії лишається лише `.env.example`.

## Локальний запуск

```bash
nvm use 22
npm install
npx prisma generate
npx prisma migrate deploy
npm run db:seed
npm run dev
```

Застосунок: [http://localhost:3000](http://localhost:3000)

### Тестові облікові дані (після `db:seed`)

| Роль | Email | Пароль |
|------|-------|--------|
| Адміністратор | `admin@example.local` | `Admin123!` |

Після `seed` у БД також зʼявляються демонстраційні компоненти робототехніки з історією рухів залишків.

## Збірка для продакшену

```bash
nvm use 22
npm install
npx prisma generate
npm run build
```

Перевірка production-збірки локально:

```bash
npm run preview
```

Артефакт збірки: `.output/`. Для деплою на Node-сервер:

```bash
node .output/server/index.mjs
```

На хостингу потрібно задати ті самі змінні середовища, що й у `.env`, і виконати міграції БД (`npx prisma migrate deploy`) перед першим запуском.

## Корисні команди

```bash
npm run dev          # режим розробки
npm run build        # production-збірка
npm run preview      # перегляд збірки
npm run db:seed      # наповнення БД демо-даними
npx prisma studio    # GUI для перегляду БД
```

## Примітки

- Кошик на сторінці `/cart` зберігається **лише в памʼяті браузера** — після перезавантаження сторінки він очищується.
- AI-функції потребують валідного `NUXT_GEMINI_API_KEY`; при лімітах API застосунок автоматично перемикається на резервну модель.
- Для production обовʼязково встановіть надійний `NUXT_SESSION_SECRET` і не публікуйте секрети в репозиторії.
